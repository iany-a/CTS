Flyweight - keyword "state" + reduce memory usage



state = set of its values



Objects have common attributes, but some of their states vary - the memory can be minimised by sharing their fixed state.



set the common attribute(s) as their own object, which will store its data and propagate to all objects created (by pointer / reference)



Flyweight Factory class will handle the common attributes.



Flyweight is like a singleton registry using a factory





Proxy pattern = close to something, in proximity



proxy class must implement or extend the object class + a reference to the initial entity (has-a)



4 types of proxies:

**virtual proxies**

**protectional / control proxies**

remote proxies

smart references proxies



smart pointers are proxy patterns



\*proxy, adapter and decorator are wrapper classes
